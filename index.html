⚠️ 오류 처리 & 방어 로직 가이드

인지도 조사를 완료하지 않으면 절대 수강완료 처리되지 않도록 아래 모든 시나리오를 반드시 방어해야 합니다.

🚨 시나리오 1: 설문 도중 페이지 이탈 (뒤로가기/다른 링크 클릭)
문제: 사용자가 설문 도중 사이드바 메뉴, 브라우저 뒤로가기, 상단 메뉴 클릭 시 응답이 유실되고 수료 불가 상태로 남음
방어:
  • beforeunload 이벤트: "인지도 조사를 완료해야 수료가 가능합니다. 정말 나가시겠습니까?" 확인 팝업
  • popstate (뒤로가기) 감지: 히스토리 조작으로 이탈 차단
  • 사이드바/상단 메뉴 링크에 onclick 가드: 설문 진행 중이면 확인 모달 표시
💡 UI 안내문구 (설문 상단에 고정):
"⚠️ 인지도 조사를 완료해야 교육 수료가 가능합니다. 페이지를 벗어나면 응답이 저장되지 않습니다."
🚨 시나리오 2: 브라우저/탭 강제 종료
문제: 브라우저 닫기, 기기 종료, 앱 전환(모바일) 시 응답이 날아감
방어:
  • 각 문항 선택 시마다 localStorage에 임시 저장 (key: survey_draft_{userId}_{year})
  • 재진입 시 localStorage에서 복원: "이전에 작성 중이던 응답이 있습니다. 이어서 하시겠습니까?"
  • 제출 성공 시에만 localStorage 삭제
🚨 시나리오 3: 네트워크 오류 / 서버 오류 (제출 실패)
문제: "제출하기" 클릭했으나 네트워크 끊김/서버 500 에러로 저장 실패. 사용자는 완료된 줄 알고 나감
방어:
  • 제출 API 응답이 200 OK + success: true일 때만 수료 처리로 이동
  • 실패 시: "제출에 실패했습니다. 다시 시도해주세요." 에러 토스트 + 재시도 버튼 표시
  • 제출 버튼 클릭 후 로딩 스피너 표시 + 중복 클릭 방지 (disabled)
  • 3회 연속 실패 시: "네트워크 상태를 확인 후 다시 시도해주세요" + 고객센터 안내
⚡ 시나리오 4: 세션 만료 (장시간 방치)
문제: 설문 페이지 열어놓고 30분+ 방치 → 세션 만료 → 제출 시 401/403 에러
방어:
  • 제출 시 401/403 응답이면: "세션이 만료되었습니다. 로그인 후 다시 시도해주세요." + 로그인 페이지 이동
  • localStorage에 임시 저장된 응답이 있으므로 로그인 후 복원 가능
⚡ 시나리오 5: 중복 제출 / 새로고침
문제: 느린 네트워크에서 제출 버튼 연타, 또는 제출 직후 새로고침하여 중복 데이터 생성
방어:
  • 프론트: 제출 클릭 즉시 버튼 disabled + 로딩 스피너
  • 백엔드: SurveyResponse에 unique_together = ['user', 'year'] → DB 레벨 중복 방지
  • 이미 존재하면 에러 대신 "이미 제출 완료" 안내 → 수료 화면으로 이동
⚡ 시나리오 6: 미응답 문항 제출 시도
문제: 일부 문항에 응답하지 않고 제출 시도
방어:
  • 모든 5문항 응답 전까지 "제출하기" 버튼 비활성화(disabled) 상태 유지
  • 미응답 문항에 빨간 테두리 + "응답해주세요" 표시
  • 백엔드에서도 5문항 모두 있는지 재검증 (프론트만 믿지 않기)
📱 시나리오 7: 모바일 특수 상황
문제: 전화 수신, 다른 앱 전환, 화면 잠금 시 페이지 상태 유실 (iOS Safari 특히)
방어:
  • visibilitychange 이벤트: 탭 비활성 시 localStorage에 자동 임시 저장
  • pagehide / freeze 이벤트 감지 (iOS Safari 대응)
  • 페이지 복귀 시 세션 유효성 체크 → 만료 시 자동 저장분 복원 안내
🔒 시나리오 8: 수료 처리 무결성
문제: 인지도 조사를 우회하고 수료 API를 직접 호출하여 수료 처리되는 경우
방어:
  • 수료 처리 API에서 반드시 체크: SurveyResponse.objects.filter(user=user, year=2026).exists()
  • False이면 수료 거부 + "인지도 조사를 먼저 완료해주세요" 응답
  • 프론트뿐 아니라 백엔드에서 이중으로 검증 (프론트 조작 방지)
